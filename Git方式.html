创建本地的git仓库

进入需要操作的文件夹打开git命令窗口

命令 说明
git init 初始化本地git库
git status 查看提交的状态
git add . 提交到暂存区
git commit -m “备注信息” 提交到历史区

本地仓库内容提交到远程仓库

远程仓库地址 ：https://github.com/Leon199254/201804JS.git

① 关联本地仓库和远程仓库
git remote add leon(远程仓库名) https://github.com/Leon199254/201804JS.git
git remote -v 查看关联的远程仓库
git remote rm leon 删除关联的远程仓库

② 推送本地仓库内容到远程仓库
git push origin master 推送到远程仓库的主分支

复习整个过程

git config credential.helper ” 清除配置的用户信息
git config –global user.name “Leon199254”
git config –global user.email “981059792@qq.com”

  1. 进入文件夹 打开git命令窗口 -> git init 初始化本地git仓库
  2. git add . 提交到暂存区
  3. git commit -m “备注提交的信息” 提交到历史区
  4. git status 查看提交状态
  5. 去github新建仓库 点+号->new respository->create respository
    会生成一个远程仓库地址 https://github.com/Leon199254/201804JS.git
  6. 进入本地仓库文件夹 关联本地仓库和远程仓库
    git remote add origin https://github.com/Leon199254/201804JS.git
    git remote -v 查看关联的远程仓库是哪个
  7. git push origin master 把本地仓库内容推送到远程仓库
    - 以后推的时候先拉取一下，养成好的习惯
  8. 若遇到远程仓库和本地仓库不一样时
    先需要把远程仓库内容拉取下
    git pull origin master –allow-unrelated-histories 强制先合并远程仓库和本地仓库的内容。

另一种提交本地项目的方式（常用）

git clone https://github.com/pengiff/2018JS.git
相当于做了三件事
- 1.建了一个本地git仓库，仓库名和远程仓库名一样
- 2.本地仓库和远程仓库关联起来了
- 3.远程仓库的内容已经拉取到本地仓库
git add .
git commit -m”“”
git push origin master

远程仓库内容和本地仓库内容不一样时，需要两个仓库保持同步后，才能往远程仓库推送内容，同步的做法就是git pull origin master 把远程仓库内容先拉取到本地仓库
